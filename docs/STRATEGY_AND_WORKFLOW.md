## 1. 투자 전략

### 1.1 페이퍼 트레이딩 전략 (4가지)

#### 전략 A: 모멘텀 (Momentum)

```
[개념]
상승 추세 + 호재(뉴스/공시)가 있는 종목의 추가 상승을 노림

[선정 조건]
- 전일 상승세 (+1% 이상)
- 뉴스 긍정 점수 15점 이상
- 또는 공시 점수 20점 이상
- 거래대금 100억원 이상

[점수 배점]
- 공시: 40점 (실적 40점, 계약 35점, 투자 30점, 사업 25점, 기술 20점)
- 뉴스: 25점 (언급 횟수 + 긍정도)
- 테마: 15점 (AI, 반도체, 2차전지 등)

[매매 규칙]
- 진입: 시초가 (09:00~09:05)
- 익절: +5% 도달 시 즉시 매도
- 손절: -3% 도달 시 즉시 매도
- 청산: 15:20까지 미도달 시 종가 청산
```

#### 전략 B: 대형주 역추세 (Large-Cap Contrarian)

```
[개념]
대형주는 일시적 하락 후 평균 회귀 경향이 있음
기관/외국인의 저가 매수로 반등 지지

[선정 조건]
- 가격: 50,000원 이상 (대형주)
- 전일 등락률: -1% 이상 하락
- 거래대금: 500억원 이상

[점수 배점]
- 하락폭: 35점 (-5% 이상 시 만점)
- 거래대금: 25점 (3천억 이상 시 만점)
- 시가총액: 15점 (10조 이상 시 만점)
- 거래량 변화: 15점 (평균 대비 2배 시 만점)
- 가격대 적합성: 10점 (5~10만원이 최적)

[매매 규칙]
- 진입: 시초가 매수
- 익절: +5% 도달
- 손절: -3% 도달
- 청산: 14:30까지 미도달 시 종가 청산
```

#### 전략 C: DART 공시 기반

```
[개념]
전일 장 마감 후 ~ 당일 장 시작 전 공시 발표 종목 매매

[공시 카테고리 점수]
┌─────────────┬──────┬──────────────────────────────┐
│ 카테고리     │ 점수 │ 키워드                        │
├─────────────┼──────┼──────────────────────────────┤
│ 실적 개선   │ 40점 │ 영업이익, 실적 개선, 흑자전환   │
│ 대규모 계약 │ 35점 │ 공급계약, 수주, MOU            │
│ 투자 유치   │ 30점 │ 투자 유치, 증자, 자금 조달      │
│ 사업 확장   │ 25점 │ 신규 사업, 해외 진출, 설립      │
│ 기술 개발   │ 20점 │ 특허, 인증, 기술 개발          │
└─────────────┴──────┴──────────────────────────────┘

[수집 시간]
- 전일 18:00 ~ 당일 08:30 공시 수집

[매매 규칙]
- 공시 점수 30점 이상 시 매수 대상
- 익절 +5% / 손절 -3%
```

#### 전략 D: 테마/정책 기반

```
[개념]
정책 수혜, 시장 트렌드 테마 관련주 매매

[테마 카테고리]
┌────────────┬──────────────────────────────────────┐
│ 테마       │ 키워드                                │
├────────────┼──────────────────────────────────────┤
│ AI        │ 인공지능, ChatGPT, 생성형AI, LLM      │
│ 반도체     │ HBM, 파운드리, 메모리, GPU           │
│ 2차전지   │ 배터리, EV, 양극재, 음극재            │
│ 바이오     │ 신약, 임상, FDA, 치료제              │
│ 방산      │ 국방, 방위산업, 무기, 미사일          │
│ 엔터      │ 엔터테인먼트, 콘텐츠, 아이돌          │
│ 게임      │ 게임, e스포츠, 메타버스              │
│ 수소      │ 수소, 연료전지, 그린                 │
│ 로봇      │ 로봇, 자동화, 드론                   │
└────────────┴──────────────────────────────────────┘

[매매 규칙]
- 테마 점수 10점 이상 + 뉴스 언급 3회 이상
- 익절 +5% / 손절 -3%
```

### 1.2 BNF 낙폭과대 전략

#### 전략 개요

```
[BNF 스타일]
일본의 전설적 개인 투자자 BNF(코테가와 타카시)의 역추세 전략
급락 후 반등을 노리는 낙폭과대 매매

[핵심 원리]
- 과매도 상태에서 기술적 반등 기대
- 대형주는 급락 후에도 펀더멘털 유지
- 공포에 매수, 탐욕에 매도
```

#### 종목 선정 기준

```
[선정 조건] - 하나라도 충족 시 대상
┌────────────────┬────────────┬───────────────────────┐
│ 기준           │ 조건       │ 설명                  │
├────────────────┼────────────┼───────────────────────┤
│ 5일 낙폭       │ -15% 이상  │ 최근 5거래일 급락     │
│ 10일 낙폭      │ -20% 이상  │ 최근 10거래일 급락    │
│ 고점 대비 낙폭 │ -25% 이상  │ 20일 고점 대비 하락   │
└────────────────┴────────────┴───────────────────────┘

[추가 필터]
- KOSPI 200 구성 종목 (대형주)
- 거래대금 500억원 이상
- 악재성 뉴스 제외 (분식, 상폐, 소송 등)
```

#### 분할 매수 전략

```
[3단계 분할 매수]
┌──────┬────────────┬────────┬─────────────────────────┐
│ 단계 │ 진입 조건  │ 비중   │ 설명                    │
├──────┼────────────┼────────┼─────────────────────────┤
│ 1차  │ -5% 하락   │ 30%    │ 초기 진입              │
│ 2차  │ -7% 하락   │ 40%    │ 추가 하락 시 물타기    │
│ 3차  │ -10% 하락  │ 30%    │ 최종 진입 (평단가 하향) │
└──────┴────────────┴────────┴─────────────────────────┘

[예시] 100만원 투자
- 1차: 30만원 (현재가 -5%)
- 2차: 40만원 (현재가 -7%)
- 3차: 30만원 (현재가 -10%)
→ 평균 단가: 약 -7.3%
```

#### 청산 전략

```
[트레일링 스탑]
- 기준: 보유 중 고점 대비 -2% 이탈 시 전량 매도
- 목적: 수익 보존 + 추가 하락 방지

[분할 매도]
┌──────┬────────────┬────────┬───────────────────────┐
│ 단계 │ 청산 조건  │ 비중   │ 설명                  │
├──────┼────────────┼────────┼───────────────────────┤
│ 1차  │ +3% 반등   │ 1/3    │ 원금 회수 시작        │
│ 2차  │ +5% 반등   │ 1/3    │ 수익 실현             │
│ 3차  │ +7% 반등   │ 1/3    │ 잔량 청산             │
└──────┴────────────┴────────┴───────────────────────┘

[손절]
- 평단가 대비 -5% 시 전량 손절
- 악재 뉴스 발생 시 즉시 청산
```

```yaml

트리거: schedule, workflow_dispatch, push

흐름:
┌─────────────────────────────────────────────────────────────┐
│ 1. 개장일 체크                                              │

│    └─ morning_candidates.json에 종목 있는지 확인            │

출력 파일:
- data/morning_candidates.json
- data/morning_candidates.db
```

## 3. 데이터 플로우

### 3.1 전체 데이터 흐름

```
[데이터 수집]
    │
    ├─ DART API ────────→ 공시 데이터
    ├─ 네이버 금융 ──────→ 뉴스 데이터
    ├─ pykrx ───────────→ 시장 데이터 (가격, 거래량, 수급)
    │
    ↓
[점수 계산] (stock_screener.py)
    │
    ├─ 공시 점수 (40점)
    ├─ 뉴스 점수 (25점)
    ├─ 테마 점수 (15점)
    ├─ 거래대금 점수 (15점)
    ├─ ... (총 145점)
    │
    ↓
[종목 선정]
    │
    ├─ morning_candidates.json (상위 5개)
    │
    ↓
[장중 모니터링]
    │
    ├─ 시초가 기록
    ├─ 익절/손절 체크
    ├─ 종가 기록
    │
    ↓
[결과 분석]
    │
    ├─ result_{date}.json (일별 결과)
    ├─ results.json (누적 결과)
    │
    ↓
[대시보드 표시]
    │
    └─ index.html (GitHub Pages)
```


## 4. 시간별 실행 흐름

### 4.1 일간 타임라인

```
┌───────┬───────────────────────────────────────────────────────┐
│ 시간  │ 이벤트                                                │
├───────┼───────────────────────────────────────────────────────┤
│ 08:10 │ [Morning Scan] 145점 종목 선정                        │
│       │   → morning_candidates.json 생성                     │
├───────┼───────────────────────────────────────────────────────┤
│ 08:50 │ [Paper Trading Select] 페이퍼 종목 확정              │
│       │   → candidates_{date}.json 생성                      │
├───────┼───────────────────────────────────────────────────────┤
│ 09:00 │ 장 시작 - 시초가 매수                                │
├───────┼───────────────────────────────────────────────────────┤
│ 09:10 │ [BNF Simulation] BNF 시뮬레이션 시작                 │
├───────┼───────────────────────────────────────────────────────┤
│ 09:00 │ 장중 모니터링                                        │
│  ~    │   - 익절 +5% 체크                                    │
│ 15:20 │   - 손절 -3% 체크                                    │
├───────┼───────────────────────────────────────────────────────┤
│ 15:20 │ 종가 청산 (미도달 종목)                              │
├───────┼───────────────────────────────────────────────────────┤
│ 15:30 │ [Paper Trading Check] 장중 결과 체크                 │
│       │   → status_{date}.json 업데이트                      │
├───────┼───────────────────────────────────────────────────────┤
│ 16:10 │ [Paper Trading Final] 최종 결과 집계                 │

│       │   → intraday_{date}.json 생성                        │
```
